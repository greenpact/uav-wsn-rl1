System Model and RL-based UAV-Assisted WSN Routing Protocol

1. Network Model
We consider a UAV-assisted wireless sensor network (WSN) consisting of N = 500 homogeneous sensor nodes randomly and uniformly deployed over a square area of 1000 m × 1000 m. Each node is initialized with energy E0 and is equipped with limited buffer capacity B. Nodes generate sensing data periodically and forward data packets in a multi-hop manner toward temporary UAV contact regions.

A single base station (BS) is located far from the sensing field and is unreachable by direct sensor transmissions. One or multiple UAVs periodically depart from the BS, visit the sensing field, collect data from sensor nodes, and return to the BS. The UAV is also energy constrained and operates under a limited mission duration.

2. Communication and Energy Model
All sensor nodes use identical radio interfaces with a fixed transmission range R. The energy consumption model follows the classical first-order radio model. The energy required to transmit a k-bit packet over distance d is given by:

E_tx(k,d) = E_elec * k + E_amp * k * d^2

The energy for receiving a packet is:

E_rx(k) = E_elec * k

3. Traffic and Buffer Model
Each node generates sensing packets according to a periodic process. Each node maintains a FIFO buffer of size B. Packet loss occurs when buffer overflow happens.

4. UAV Contact and Timing Model
The UAV follows a bounded dynamic trajectory starting and ending at the BS. During each mission, the UAV broadcasts its predicted next-contact schedule to nodes within communication range. Each node maintains an estimate of the remaining time to UAV contact.

5. State Abstraction for Routing
Each node uses a compact local state representation:

s = [remaining energy ratio, buffer occupancy ratio, neighbor density, link quality class, predicted UAV contact time]

6. Action Space
The routing action is the selection of one next-hop neighbor among the K best reachable neighbors.

7. Reward Model
The immediate reward for forwarding a packet is defined as:

r = w1 * successful_delivery − w2 * energy_consumption − w3 * queuing_delay − w4 * packet_drop

8. Offline Centralized Training Model
A digital twin of the network is implemented at the BS using OMNeT++ 6.3.0 and INET 4.5. The environment includes randomized deployments, traffic patterns, radio conditions, and UAV schedules. A centralized deep Q-learning agent learns a shared routing policy using the abstract state representation.

9. Online Lightweight Adaptation
During real operation, the learned policy is frozen. Each node only maintains a lightweight neighbor bias table to compensate for local and short-term variations. This avoids online exploration and convergence delay.

10. Optimization Objectives
The routing protocol aims to minimize energy consumption and end-to-end delay, maximize packet delivery ratio, and extend network lifetime measured by the first-node-death round and overall network survival time.

References:
Heinzelman et al., LEACH: An Energy-Efficient Communication Protocol for Wireless Microsensor Networks.
Boyan and Littman, Packet routing in dynamically changing networks: A reinforcement learning approach.
