scientifically valid dqn training data generation workflow:
    Since training a DQN directly inside a discrete-event simulator is computationally prohibitive, the "Gold Standard" is the Offline-to-Online Validation Workflow.

    1. The workflow architecture:
        Phase I: Trace Collection - using omnet++, Run a "dumb" simulation with GGM mobility to log realistic radio/energy data.
        Phase II: RL Training - using Python, Train the DQN using the traces/agents to generate the Optimized Q-Table.
        Phase III: Protocol Testing - using the simulation protocol designed (omnet++ (C++)), Hardcode the Q-Table/Weights or the ANN model into your custom routing module for final results.
    2. Detailed Data Generation Steps:
        Step A: Environmental Setup (OMNeT++)
            Set up the 1000x1000m grid with 400 nodes and 1 UAV using the Gauss-Markov Mobility module.
            Action: Run the simulation for T hours.
            Output: Export a .csv log every time a packet is generated or a node moves.
            Trace Columns: [Timestamp, NodeID, PosX, PosY, Energy, BufferCount, LinkQuality_to_Neighbors].
        Step B: Python Agent Emulation (Gymnasium)
            Create a custom WSN-Env. Instead of simulating the radio stack, use the Trace Data from Step A to "play back" the environment.
    
            Agent State Vector Formulation (S):
                1. resEnergyRatio: E_remaining/E_initial
                2. bufferOccRatio: Packets_current/Buffer_maxsize
                3. neighborDensity: Count of nodes within R_comm of node
                4. linkQualClass: Categorize based on distance or SINR (e.g., 0: Poor, 1: Good).
                5. predictedUAVContactTime: Calculated using the GGM parameters:
                    delta_t = [sqrt[square(X_uav - X_node) + square(Y_uav - Y_node)]]/v_uav
                    (Note: Since GGM has a bias alpha, use the current velocity vector to estimate arrival).
        
        Step C: Reward Function (R)
            To ensure the protocol chooses nodes that help the UAV, use a multi-objective reward:
                R = w1 * E_ratio  +  w2 * (1-Buff_ratio)  +  w3 * NodeDegree  -  w4 * Dist_uav

    3. Implementation Pseudocode (Python Data Gen)
        import gymnasium as gym
        import numpy as np

        class WSNRoutingEnv(gym.Env):
            def __init__(self, trace_data):
                self.nodes = initialize_agents(400) # From OMNeT++ trace
                self.uav = UAV_Agent(mobility="Gauss-Markov")
                
            def step(self, action):
                # action = selected next-hop node 'j'
                
                # 1. Update UAV Position via GGM Equation
                self.uav.update_pos(alpha=0.8, target_bias=self.get_dqn_bias())
                
                # 2. Calculate State for node 'i' and candidate 'j'
                state = [
                    self.nodes[i].energy_ratio,
                    self.nodes[i].buffer_ratio,
                    self.nodes[i].neighbor_density,
                    self.calculate_link_qual(i, j),
                    self.predict_uav_arrival(j),
                    self.dist_to_uav(j)
                ]
                
                # 3. Calculate Reward
                reward = self.compute_reward(state)
                
                return np.array(state), reward, done, info

    4. Transition to OMNeT++ C++ Module
        Once training is done, you export the Q-table or the Neural Network weights (if using a lightweight MLP). In your OMNeT++ C++ Routing Module:
            On handleMessage, the node looks at its local state.
            It performs a Forward Pass of the DQN (using a C++ math library or simple Matrix multiplication).
            The node selects the neighbor with the highest Q-value as the next hop.
